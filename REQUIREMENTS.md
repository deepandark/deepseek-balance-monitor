# DeepSeek 余额监控器 — 需求文档

## 1. 项目概述

一款桌面应用，通过 DeepSeek 官方 API 定期查询账户余额，本地存储历史记录，
通过余额差值反推每日用量，并以图表形式展示趋势。

---

## 2. 功能需求

### 2.1 余额查询
- 调用 DeepSeek 官方 API：`GET https://api.deepseek.com/user/balance`
- 认证方式：Bearer Token（用户填写的 API Key）
- 展示字段：
  - 总余额（total_balance）
  - 充值余额（topped_up_balance）
  - 赠送余额（granted_balance）
  - 货币类型（CNY）
  - 账户可用状态（is_available）

### 2.2 用量推算
- 不依赖 DeepSeek 用量 API（无官方接口）
- 计算公式：`消耗 ≈ 前一次记录的余额 - 本次记录的余额`
- 当余额增加时（用户充值），自动识别为充值事件，不纳入消耗统计
- 展示维度：今日消耗、昨日消耗、近 7 天、近 30 天

### 2.3 历史趋势
- 折线图：余额随时间变化（总余额 / 充值余额 / 赠送余额三根线）
- 柱状图：每日消耗量
- 时间范围切换：7 天 / 30 天 / 90 天

### 2.4 后台轮询
- 系统托盘驻留，关闭窗口不退出程序
- 可配置轮询间隔（默认 60 分钟）
- 轮询失败重试 3 次，静默处理（无弹窗打扰）

### 2.5 配置管理
- API Key 设置
- 轮询间隔设置
- 开机自启开关

### 2.6 不做的
- ❌ 告警通知
- ❌ 多账户
- ❌ 用量/费用 API 调用（无官方接口）

## 3. 技术方案

| 层 | 选择 | 理由 |
|---|---|---|
| 桌面框架 | Electron 28+ | 生态成熟，托盘/开机启动现成方案 |
| 前端 | React 18 + TypeScript | 类型安全，生态丰富 |
| UI 框架 | Tailwind CSS + shadcn/ui | 简洁现代，组件即复制 |
| 图表库 | ECharts (echarts-for-react) | 中文友好，交互丰富 |
| 本地数据库 | better-sqlite3 | 同步 API，无需 ORM，轻量 |
| HTTP 请求 | fetch (Node 18 内置) | 零依赖 |
| 构建工具 | electron-vite | Vite 速度，Electron 专用 |
| 包管理 | pnpm | 快，磁盘友好 |

## 4. 数据模型

### 表：balance_records
| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增 |
| total_balance | REAL | 总余额 |
| topped_up_balance | REAL | 充值余额 |
| granted_balance | REAL | 赠送余额 |
| currency | TEXT | 货币类型 |
| is_available | INTEGER | 账户可用 |
| raw_json | TEXT | 原始响应 JSON（备用） |
| created_at | TEXT | ISO 时间戳 |

### 推导字段（查询计算，不存储）
- `daily_consumption = prev.total_balance - curr.total_balance`（正数表示消耗）
- 若差值为负，标记为充值事件

## 7. 验收标准

1. 填入 API Key 后点击刷新，能显示当前余额
2. 挂机 2 小时后，能看到 2 条历史记录
3. 图表正确展示余额曲线和每日消耗柱状图
4. 关闭窗口后程序在托盘中继续轮询
5. 双击托盘图标恢复窗口
